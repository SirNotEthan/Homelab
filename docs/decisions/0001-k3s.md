# ADR-0001: Use k3s as the Kubernetes distribution

- Status: Accepted
- Date: 2026-07-03

## Context

The cluster runs on low-power ThinkCentre hardware with modest memory. It needs
a conformant Kubernetes platform without the operational overhead of assembling
and maintaining every control-plane component independently.

## Decision

Use k3s for the control plane and worker nodes. Begin with one server node on
`m910q-01` and three agent nodes, while preserving the option to add control-plane
nodes later.

## Consequences

- Installation and upgrades are simpler than a kubeadm-built cluster.
- The packaged defaults reduce resource use and component count.
- k3s-specific packaging and defaults must be understood during troubleshooting.
- The initial single control plane is not highly available and requires tested
  datastore backup and recovery.

## Alternatives considered

- kubeadm: more component-level control, but more maintenance for this scale.
- Docker Compose: simpler for individual hosts, but does not meet the desired
  scheduling and GitOps operating model.
