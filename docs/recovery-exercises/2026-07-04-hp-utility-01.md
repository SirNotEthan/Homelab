# hp-utility-01 adoption exercise

## Timing

- Status: Successful
- Start: 2026-07-04 23:39:15 BST (22:39:15 UTC)
- Finish: 2026-07-10
- Duration: Not measured as a continuous recovery window; work was paused
  between adoption sessions.

## Target

- Host: `hp-utility-01`
- Intended role: private DNS, local backup target, and utility desktop
- Source: existing Ubuntu Desktop 24.04 LTS installation

## Existing state

- The installation already used a 100 GB LVM root volume and retained roughly
  1.72 TB free in its volume group.
- Grafana, Prometheus, and a manually installed Node Exporter were experimental
  and contained no data that required preservation.
- The `server` account contained no files that required preservation.

## Interventions

- Package cleanup initially waited for an active unattended-upgrades process;
  the package-manager lock was left intact and allowed to clear normally.
- Experimental Grafana, Prometheus, and Node Exporter components were removed
  before the host was brought under repository management.
- A dedicated administrative account was created and granted sudo access.
- The host was renamed to `hp-utility-01`.
- Ethernet was moved from DHCP to the reserved static host address.
- The shared Ubuntu baseline was applied while preserving the graphical
  desktop target.
- Tailscale was installed and the host was joined to the tailnet.
- The root filesystem was intentionally left at 100 GB so the remaining volume
  group space can be allocated later for backup storage.

## Validation

- Ubuntu Desktop 24.04 LTS is running with the updated kernel.
- UEFI boot was verified.
- Secure Boot is currently disabled and should be enabled in firmware during a
  later physical maintenance pass.
- The root filesystem remains at 100 GB with roughly 1.72 TB free in the
  volume group.
- Static Ethernet, DNS, and the default route work.
- The Ansible baseline and Tailscale role are idempotent.
- SSH, chrony, and Tailscale services are active.
- LAN and Tailscale SSH connectivity work from the administrative workstation.
- Disk SMART health passed.
- The system remains in `graphical.target` for utility desktop use.
