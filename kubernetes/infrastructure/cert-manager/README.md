# cert-manager

cert-manager issues and renews the private application wildcard certificate
using Let's Encrypt DNS-01 validation through Cloudflare.

## Secret handling

The Cloudflare API token Secret is intentionally not stored in Git as a plain
Kubernetes Secret. It is currently created manually in the `cert-manager`
namespace and should be migrated to a SealedSecret.

Required Secret name and key:

```text
cloudflare-api-token-secret
└── api-token
```

The ACME account key Secrets and issued TLS Secrets are generated by
cert-manager and must not be committed or manually sealed.

## Managed resources

This directory manages:

- Let's Encrypt staging and production `ClusterIssuer` resources;
- staging and production wildcard `Certificate` resources for the private
  application namespace.

## Validation

```bash
kubectl get pods -n cert-manager -o wide
kubectl get clusterissuer
kubectl get certificate -n ingress-system
kubectl get application cert-manager-config -n argocd
```
